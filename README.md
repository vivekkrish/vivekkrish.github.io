# Vivek Krishnakumar — Personal Website

Personal portfolio hosted at [vivekkrish.com](https://vivekkrish.com), built as a dependency-free static SPA with automated citation and publication syncing via GitHub Actions.

---

## Stack

| Layer | Technology |
|---|---|
| Structure | HTML5 (single-page, hash-routed) |
| Styling | Vanilla CSS3 (custom properties, responsive grid, dark/light theme) |
| Logic | Vanilla JavaScript ES6 (no frameworks) |
| Publications | Custom static build (populated from JSON generated by the scraper script) |
| Citation Stats | Google Scholar with Semantic Scholar API fallback (stored in `assets/resources/citation_metrics.json`) |
| Hosting | GitHub Pages (auto-deploy on push to `master`) |
| Automation | GitHub Actions (daily Scholar sync cron) |

## Configuration (`assets/resources/profile_config.json`)

Edit this file to update site content without touching any HTML or JS:

```json
{
  "name": "Vivek Krishnakumar",
  "title": "Associate Director of Bioinformatics",
  "bio": "...",
  "orcid": "0000-0002-5227-0200",
  "email": "hello@vivekkrish.com",
  "scholar": "cLlRcPYAAAAJ",
  "semanticScholar": "34901477",
  "featuredPublications": [
    "The Medicago genome provides insight into the evolution of rhizobial symbioses",
    "Araport11: a complete reannotation of the Arabidopsis thaliana reference genome",
    "JCVI: A versatile toolkit for comparative genomics analysis"
  ],
  "socials": {
    "github": "vivekkrish",
    "linkedin": "vivekkrish",
    "speakerdeck": "vivekkrish",
    "scholar": "cLlRcPYAAAAJ",
    "instagram": "vivekkrish",
    "twitter": "vivekkrish"
  },
  "socialLinksOrder": ["github", "linkedin", "speakerdeck", "scholar", "instagram", "twitter", "email"]
}
```

| Key | Purpose |
|---|---|
| `scholar` | Google Scholar profile ID — used to scrape citation counts and your publication list |
| `semanticScholar` | Semantic Scholar Author ID — used as a stable API-based fallback if Google Scholar scraping gets blocked |
| `orcid` | ORCID profile ID (for record-keeping/reference) |
| `socialLinksOrder` | Controls which icons appear on the homepage and in what order |

> **Note:** Mendeley and BibBase integrations are no longer used. Publications and citation stats are synced directly from Google Scholar (with a Semantic Scholar API fallback) and stored locally in `assets/resources/citation_metrics.json` and `assets/resources/publications_list.json`.

## Citation & Publication Automation

Citation stats, the annual citation chart, and the publication bibliography list are kept up-to-date automatically — no manual edits needed.

### How it works

1. **`scripts/update_scholar.py`** — Reads the configurations from `assets/resources/profile_config.json` and performs the sync:
   - **Step 1: Google Scholar Scrape**: It attempts to fetch and parse your profile page (`scholar.google.com/citations?user=<id>`) using browser-realistic headers and regional domain rotation. If successful, it extracts:
     - Total citations, h-index, i10-index, and paper count (written to `assets/resources/citation_metrics.json`)
     - Per-year citation history (written to `assets/resources/citation_metrics.json`)
     - Scraped publications metadata list (written to `assets/resources/publications_list.json`)
   - **Step 2: Semantic Scholar Fallback**: If Google Scholar blocks the request (e.g. `403 Forbidden` on all domains), the script automatically queries the official Semantic Scholar API using your `semanticScholar` ID. It updates the stats and publications, while preserving the existing year-by-year citation history from the local cache.

2. **`.github/workflows/update_scholar.yml`** — A GitHub Actions workflow that:
   - Runs on an **off-peak daily cron schedule** (4:37 UTC) to avoid hourly GHA runner concurrency blocks.
   - Can also be triggered manually via `workflow_dispatch`.
   - Checks out the repo, runs the update script, and commits/pushes updates to `assets/resources/citation_metrics.json` and `assets/resources/publications_list.json` back to `master`.

### Running the sync script manually

No external packages are required (uses Python's standard `urllib` and `re` libraries):

```bash
python scripts/update_scholar.py
```

## Local Development

The site fetches `assets/resources/config.json` at runtime, so it must be served through a local HTTP server (browsers block `file://` cross-origin reads).

```bash
# Python (built-in)
python -m http.server 8000

# Node.js
npx http-server -p 8000
```

Then open `http://localhost:8000`.
